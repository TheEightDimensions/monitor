# BAC-OMC-DPM-MULT-FLOW 气象监控业务系统分析报告

## 一、项目概述

这是一个**气象监控业务系统**（BAC-OMC-DPM-MULT-FLOW），为山东省泰安市气象局构建，用于监控气象数据流程和基础服务器资源。项目已完成前后端分离的现代化重构。

## 三、是否有实际业务功能？

**有完整的业务框架，但当前运行在全 Mock 数据上。**

项目的业务逻辑结构是真实的，包括：

| 模块 | 功能 | 数据来源 |
|------|------|----------|
| 数据流程监视 | 采集→传输→处理→同步→服务 + 全流程告警 | Mock JSON |
| 基础资源监视 | 磁盘/内存/网络IO/CPU + 阈值告警 + 短信通知 | Mock JSON |
| GIS 站点展示 | 泰安市气象站点地图 | Mock JSON |
| 综合看板 | 系统评分、站点分布、数据类型分布、传输状态 | Mock JSON |

DI（数据流程完整性监视）和 EI（异常告警）日志格式都已规范实现，但只输出到控制台。阈值告警逻辑（`_check_threshold`）已可用，触发后调用 `alert_and_notify`，但短信发送也只是控制台打印。

## 四、如何对接实际业务数据

项目已预留了清晰的对接入口，按优先级排列：

### 1. 改造 `core/data_io.py`（核心切换点）

当前所有 service 都通过 `read_local_json()` 获取数据。需要实现两个预留的空方法：

```python
# 已预留的数据库查询桩
def query_database(sql: str):
    pass

# 已预留的外部 API 调用桩
def fetch_external_api(url: str):
    pass
```

### 2. 对接数据库（`config.ini` 的 `[db]` 区已有配置）

```ini
[db]
host = 10.78.200.254
user = super
pwd = taqx9210
db = alarm
table_d = dsta      # 区域站
table_n = nsta      # 国家站
```

实现示例：

```python
import pymysql

def query_database(sql: str):
    conn = pymysql.connect(
        host=settings.DB_HOST, user=settings.DB_USER,
        password=settings.DB_PWD, database=settings.DB_NAME
    )
    try:
        with conn.cursor(pymysql.cursors.DictCursor) as cur:
            cur.execute(sql)
            return cur.fetchall()
    finally:
        conn.close()
```

### 3. 对接天镜/MUSIC 气象数据接口（`[tq]` 区已配置）

```ini
[tq]
serviceIp = 10.76.90.120
userId = USR_WJP
interfaceID = getSurfEleByTimeAndStaID
datacode_h = SURF_CHN_MUL_HOR    # 逐小时
datacode_m = SURF_CHN_MAIN_MIN   # 逐分钟
```

这是 CIMISS/MUSIC 数据服务接口，用于获取实际气象观测数据（到报率、及时率等）。

### 4. 对接短信告警（`[msg]` 区已配置）

```ini
[msg]
url = https://112.35.10.201:28888/sms/submit
ecName = 山东省泰安市气象局
```

修改 `core/notification.py` 中的 `send_sms()`，将控制台打印替换为 HTTP 调用该短信网关。

### 5. 对接天镜日志查询（`[tj]` 区已配置）

```ini
[tj]
url = http://10.76.89.185/qxjapi/logs/querylogs
region = 3709     # 泰安区号
```

用于查询数据传输日志，获取到报/缺报的实际状态。

### 6. 对接 DI/EI 日志推送

当前 `monitor.py` 中的 DI/EI 消息只是 `print()` 输出。对接时需替换为向天镜平台发送 HTTP POST：

```python
import requests

def send_di_log(fields: dict):
    di_msg = { ... }  # 现有构造逻辑
    requests.post("http://<天镜地址>/api/di", json=di_msg)
```

## 五、对接推荐路径

1. **先跑通 Mock** — 确保前后端联调正常
2. **接数据库** — 改 `data_io.py`，替换 monitor/resource 的 Mock 读取
3. **接 CIMISS/MUSIC** — 让气象站点数据变为实时
4. **接短信网关** — 让告警真正触达
5. **接 DI/EI 日志** — 让监控数据回传天镜

每一步只需修改 `data_io.py` 和 `notification.py`，业务 service 层代码无需改动——这正是项目"高内聚、低耦合"设计的价值。
